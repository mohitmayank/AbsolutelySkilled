## Overview

software-engineering is the default skill for any software or code work. It merges two stages into one adaptive lifecycle. **DESIGN** runs a codebase-first, ultrathink-powered interview that turns a vague ask into a validated spec a staff engineer would approve. **BUILD** decomposes that spec into a dependency-graphed task DAG, runs independent tasks in parallel waves, enforces test-first development, and verifies every task with generator-evaluator separation.

The depth of both stages scales to the task. A complexity triage routes each request: trivial fixes take a light path (confirm the plan, implement, prove it with a test), while complex work - greenfield projects, migrations, cross-cutting features - runs the full design -> decompose -> parallel-execute -> verify -> converge pipeline. Progress is tracked on a persistent `.software-engineering/board.md` that survives across sessions.

Key capabilities:
- **Adaptive depth** - one pipeline that scales rigor from a one-line fix to a full migration
- **Codebase-first intelligence** - searches the code before asking the user any answerable question
- **Dependency-graphed parallel waves** - independent sub-tasks execute simultaneously, respecting the DAG
- **Generator-Evaluator verification** - a separate evaluator agent grades each task against a scored rubric
- **Persistent board** - resume across sessions instead of restarting

This skill is the merge of the `absolute-brainstorm` (design) and `absolute-human` (execution) skills.
